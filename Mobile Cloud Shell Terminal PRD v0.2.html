Mobile Cloud Shell Terminal PRD v0.2

iPhone 세로모드에서 Google Cloud Shell에 SSH로 직접 접속하고, Codex/Hermes agent 작업을 tmux Workspace 단위로 안정적으로 이어가는 전용 터미널 앱.

Native SSHCloud Shell APItmux WorkspaceiPhone PortraitLLM-readable

0. 핵심 요약

이 앱은 Cloud Shell Editor나 WebView 앱이 아니다. Google Cloud Shell API로 environment를 시작하고 SSH로 직접 접속하는 네이티브 터미널이다. 앱 시작 시 살아있는 mobile-agent_* tmux 세션을 보여주고 사용자가 선택한다. 없으면 “새로 열기”를 보여준다.

터미널 PTY는 사용자 입력 전용이다. 세션 목록 조회, 세션 생성, kill, metadata 저장은 별도 management exec channel에서 수행한다. 이렇게 해야 Codex/Hermes TUI 입력창에 관리 명령이 섞이지 않는다.

3. Cloud Shell 연동

API목적
GetEnvironmentusers/me/environments/default 상태 및 SSH 정보 조회
StartEnvironmentCloud Shell environment 시작
AddPublicKey앱의 SSH public key 등록
AuthorizeEnvironmentCloud Shell 안에서 gcloud 수동 로그인 최소화
앱 실행 → OAuth 확인 → GetEnvironment → 필요 시 StartEnvironment
→ AddPublicKey → AuthorizeEnvironment → ssh_username/host/port 수신
→ SSH 접속 → live mobile sessions 조회

4. Workspace / tmux 세션

세션 이름 규칙

mobile-agent_YYYYMMDD-HHmm_{letter}
예: mobile-agent_20260516-1530_a
예: mobile-agent_20260516-1530_b
항목정책
letter한 글자. a-z 후 필요 시 A-Z
충돌같은 분에 이미 존재하는 letter는 건너뜀
display name사용자가 자유롭게 변경. tmux session name은 바꾸지 않음
metadata~/.mobile-cloud-shell/workspaces.json

metadata 예시

{
  "schemaVersion": 1,
  "workspaces": {
    "mobile-agent_20260516-1530_a": {
      "displayName": "Codex: iOS PRD 정리",
      "createdAt": "2026-05-16T06:30:00Z",
      "lastOpenedAt": "2026-05-16T07:01:03Z"
    }
  }
}

8. 백그라운드 / 재접속

성공 기준은 “iOS 백그라운드에서 SSH 소켓이 항상 살아있다”가 아니다. 성공 기준은 “Cloud Shell 안의 tmux Workspace가 살아있으면 앱 복귀 시 같은 Workspace에 자동 attach한다”이다.
